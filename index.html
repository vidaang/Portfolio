    Multimodal Human and Ambience Sensing System



    Background Information

    This project was created for WEAR Lab at UCF. The project was essentially replicating a smart watch and developing an app to pair with the watch. The app would connect to the watch via bluetooth and communicate the data collected from the sensors of the watch. This app would also be considered as our framework app where its contents could be replicated to create other modified versions of the app but for different devices within the lab



    Building Process

    Bluetooth Low Energy was used to connect the device to the app. A server, which utilized various APIs such as Bleak and FASTAPI, was created to host the connection, monitor connection status, and process data between the watch and app. MongoDB was used to set up the database to store user data.

    Motion 2 Sound



    Background Information

    This project was created for WEAR Lab at UCF. The purpose of this project was communicating words to medical providers through sign language. An armband device would collect the gesture from the pateint and send the motion data to a machine learning model called Long and Short Term Memory (LSTM). The model would process the data and translate it into one of the 30 gestures available. The model would then send the word to the app to be displayed to the user.



    Building Process

    This app was also built along the framework of MuHASS so it utilizes the same features including Bluetooth Low Energy and a MongoDB database. The app had unique features of its. A text-to-speech API would verbalize the word in English and a logs functionality would record all the words sent to the app by the LSTM model.

    Variable Arm Resistance Suit



    Background Information

    This project was created for WEAR Lab at UCF. The goal for this project was replicating a dumbell that patients could use to train their arm. This would be utilized for physical therapy training. The arm has sensors that would send data to an app that would display the infornmation to the user. The app would include libraries that has training protocols which would be assigned to the patient by the medical provider. The protocols would include instructions on how the patient should train and track the patient's progress.



    Building Process

    The MuHASS framework was utilized to quickly build the foundation of the app. It share similar qualities to MuHASS such as Bluetooth Low Energy Connection and a MongoDB database.
